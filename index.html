Font Properties

Although some of the properties sounds similar to text properties, think of font properties as those that will change the style of the font on the page. Its size, the family, etc.

Font Properties Covered in Your zyBook

font-family
This is where you'll define the fonts you want to use. There should be more than one in this list, and they should look similar to each other, so that if one doesn't load, another one in the list will.
font-size
Allows you to control the size of the text on your page. The browser does apply some defaults, so you should notice right away which elements on a page are headings and paragraphs (even though we don't choose tags for style).
font-weight
You can use this property to make your fonts bolder or lighter. Usually headings are more bold and text is at a normal or medium weight. Don't make your text so light or small that no one can read it!
font-style
Can be used to make fonts bold, underlined, italic, etc.
font-variant
This property specifies the variant of the font, which can be set to things like small caps and uppercase.
font
This is a shorthand property that you can use to set (in this order) a font's: style, variant, weight, size/(optional) line-height, and family all at once.

Using Web Fonts

There are a lot of ways to use fonts on the web. You can host your own files, or you can link to fonts hosted by a service like Google Fonts. Let's grab a font to use on our page for our headings. (Often, the heading font is different than the one used for the body text).

In Chrome, go to the font "Seymour One" on Google Fonts. We're going to add this to the page. Follow the steps below to use a Google Font:

  1. Choose the font you'd like to use on your site from those shown on Google Fonts, then click on the name to go to the page showing all of its styles/weights.
  2. From that page, add the style you need by clicking on the plus sign next to them. (Just choose the ones you need, the more you add, the slower your page will load).
  3. Once all of your font styles are selected and visible in the side panel, copy the link code from the sidebar, and paste it in the head of your HTML document before your link to your CSS file.
  4. To use the font in your CSS, copy the font-family rule at the bottom of the sidebar and paste it into the parts of your CSS where you want to use that font. Be sure to adjust for weights if necessary.

Font Sizing

There are a lot of units you can use to set sizes for all of the elements on your page, and that incudes fonts. However, there are some important things to keep in mind.

  1. Use relative units. If you use pixels, you can take away the user's ability to zoom into the text or make it larger with their device settings.
  2. Make sure that your body text is at least 1rem (which is about 16 pixels). This is a minimum font size for readability/accessibility.

Text Properties

Text properties do have some settings that sound similar to font properties, but think of these as those that would do more to change the way the text layout looks on the page.

Text Properties Covered in Your zyBook

text-align
This controls how the text is aligned inside of its container. Options are: left, right, center, and justified.
text-decoration
This is a shorthand that controls the color, thickness, style, and kind of line used with your text.
text-transform
You can use this property to convert your text to all upper- or lower-case letters, in addition to other settings.
text-indent
Text indent will indent the first line of your paragraph(s). This can make it easier to scan to find a new paragraph, but you can also use spacing for this.

Line Height and Width for Text - Best Practices

These two properties are important for readability and accessibility (which means that they're also important for SEO!), so be sure to follow the guidelines below.

  • Line height should be set to more than 1, 1.5 is a common setting.
  • Limiting how wide a paragraph of text is allowed to get on a screen makes it much easier to read (and looks a lot better too!). Limit your text widths to 45-90 characters (that's a bog range, generally I aim for 55-65).
  • Use the ch unit for setting a max width on text. We generally want to limit the number of characters anyway, and this makes it easy to improve readability!
